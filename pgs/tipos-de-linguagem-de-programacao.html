Quais tipos de linguagem de programação existem e quais são as suas diferenças?

Paradigmas de programação
Introdução
Antes de existirem os paradigmas de programação, os códigos eram diretamente descritos em linguagem de máquina que é composta por zeros e uns, que era extremamente complexa de programação. Por isso foram criados os paradigmas de programação, para nos ajudar na semântica(significado) do código.
Existem diferentes paradigmas de programação e cada um desses diferem muito na forma de programar. Por isso é muito importante e interessante conhecer um pouco de cada um desses tipos.
Programação Imperativa
A linguagem de programação imperativa é baseada em estados, que são as variáveis no programa, e ações que manipulam o estado dessas variáveis, os procedimentos, que são ações que mudam as variáveis. O nome do paradigma, Imperativo, está ligado ao tempo verbal imperativo, onde o programador diz ao computador: faça isso, depois isso, depois aquilo... Este paradigma de programação se destaca pela simplicidade, uma vez que todo ser humano, ao se programar, o faz imperativamente, baseado na ideia de ações e estados, quase como um programa de computador.
1 int x;
Nesta linha definimos a variável x, que tem um estado que não foi definido
2 x = 1;
Agora no caso, definimos que o estado da variável x é 1, então este é um procedimento.
Isso tudo ocorre no processamento, que são as ações e estados que ocorrem no programa.
Por usar procedimentos também é definida como Programação Procedural.
Programação Estruturada
A programação estruturada é uma disposição da programação que independente da linguagem e complexidade facilita a compreensão da solução, através de um número restrito de mecanismos.
    Os mecanismos são compostos por três estruturas:
  • Sequência: O algoritmo é executado de cima para baixo e da esquerda para direita;

    • Exemplo

    1 float x = 2; float y = 3; 3 float z = 4;
    Na linha 1, x recebeu 2, depois y recebeu 3 e na linha 2 z recebeu 4, o que há em comum é que as atribuições sempre são de cima para baixo e da esquerda para direita.
  • Decisão: A tarefa é executada após um teste lógico;
  • Iteração: A partir do teste lógico, um trecho de código pode ser repetido quantas vezes você desejar ou necessitar.
Estruturas de controle
Condicional: As instruções do programa são executadas conforme o estado lógico de uma variável e podemos testar isso com if, else, else if...etc.

Exemplo

1 int x = 1; 2 int y = 2; 3 if(x < y){ 4 printf(“x é maior que y”); 5 }
Aqui no caso será exibido no console que x é maior que y, e isso é verdade pois a condição lógica é verdadeira.
Repetição: as instruções são executadas até que um determinado estado de uma variável seja satisfeito.

Exemplo

1 while(x < 10){ 2 printf(“x ainda é maior que 10); 3 x = x + 1; 4 }

Aqui no caso, a estrutura de repetição while irá “iterar” até que x seja diferente ou maior que 10.

Recusão: aqui entramos em um tópicos um pouco mais avançado, mas resumidamente, as instruções são feitas com auto-chamadas, ou seja, funções chamam elas mesmas até que certas condições lógicas sejam satisfeitas.

Exemplo

function fatorial(x){ if(x > 1){ return x*fatorial(x-1); } }

Programação Modular

Programação modular é uma forma de programação no qual o desenvolvimento das rotinas de programação é feito através de módulos, que são interligados entre si através de uma interface comum.

Exemplo

Na linguagem Python cada arquivo .py é um módulo e eles podem ser importados para qualquer outro arquivo .py com o comando import.

Supomos que nós criamos um arquivo modulo1.py e queremos usar as funções, métodos que criamos no modulo2.py, simplesmente podemos fazer no módulo2.py

import modulo1

e pronto, nós temos todas as funções do modulo1.py

Na linguagem C podemos criar arquivos .h e .c, os .h são os arquivos de cabeçalhos que não definem o que a função faz, mas sim a assinatura, eles especificam quais funções definimos no .c, os .c é o queremos que as funções façam, definimos neste arquivo como queremos que a função, por exemplo, imprimirHelloWorld faça.

Programação Orientada a objetos

É um paradigma de programação que definem objetos, esses objetos são entidades e podem ser literalmente qualquer coisa, como um carro, uma pessoa, um livro etc.

Cada objeto tem três características principais:

  • Estado: representa os dados do objeto.
  • Comportamento: É a funcionalidade de um objeto ou função do objeto.
  • Identidade: É a identidade do objeto, um atributo específico que é diferente dos demais

Exemplo 1: estado

Em uma entidade pessoa existe os atributos nome, CPF, idade,etc...

Exemplo 2: comportamento

Uma pessoa pode correr, deitar-se, mudar de nome etc. Isso é o comportamento de um objeto pessoa.

Exemplo 3: identidade

Uma pessoa tem CPF e cada uma das pessoas têm um CPF diferentes dos demais, sendo isso sua identidade.

Programação Linear

A programação linear não é especificamente da área de tecnologia da informação em si mas sim da matemática, são problemas de otimização que possuem uma função objetivo que pode ser o lucro de um bolo e restrições como quantidade máxima de açúcar disponível do dia, horas-homem trabalhadas, etc...
